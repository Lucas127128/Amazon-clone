# AGENTS.md

## Monorepo Structure

Three main packages in `packages/`:

- **`server`**: Elysia backend (HTTP API)
- **`web`**: Vite frontend (HTML + TypeScript)
- **`shared`**: Common types, schemas, utilities
- **`testData`**: Test fixtures (imported as `"testdata"` workspace)

## Package Manager

**Use Bun, not npm or yarn.**

- Install: `bun i`
- CI installs with: `bun i --frozen-lockfile`
- Run scripts: `bun run <script>`
- Run in workspace: `bun run --cwd ./packages/<name> <script>`

## Build & Dev

| Command         | Effect                                                            |
| --------------- | ----------------------------------------------------------------- |
| `bun run build` | Builds web package (`vite build`)                                 |
| `bun run dev`   | Runs all packages in parallel (server watches, web serves)        |
| `bun run start` | Starts production server from `./packages/server/start-server.ts` |

Dev server runs HTTPS locally; `mkcert` certs at `packages/server/certs/` (required for CI).

## Testing

**Vitest with three separate projects** (configs in `vitest.config.ts:16-65`):

- **`bun`**: Node-like environment for util/data tests; includes `vitest.setup.ts`
- **`happy-dom`**: DOM environment for UI tests
- **`api`**: Integration tests against running server

Run tests:
| Command | Effect |
|---------|--------|
| `bun run test` | Runs all projects with watch mode |
| `bun run test:ci` | CI mode: no watch, all projects, server must be running |
| `bun run test:unit` | Unit tests only (skips orders, uses `bun` + `happy-dom`, no watch) |
| `bun run test:api` | API integration tests with watch |

Tests use `happy-dom` with SSL disabled (`disableStrictSSL`, `disableSameOriginPolicy`).

## Linting & Formatting

- **Lint**: `bun lint` â†’ oxlint (eslint + TypeScript plugins + oxc rules)
- **Format**: `bun fmt` â†’ oxfmt (in-place code formatting)

**Key lint rules to watch:**

- `no-barrel-file`: Barrel imports banned (`index.ts` re-exports)
- `max-lines`: 180 lines per file (strict)
- `consistent-type-definitions`: Use `type`, never `interface`
- `consistent-type-imports`: Import types as `import type`
- `simple-import-sort`: Auto-sort imports (enforced by oxfmt)
- `vitest/max-nested-describe`: Max 3 levels of `describe()` nesting
- `import/no-cycle`: Max dependency depth 3

## Pre-Commit Hooks

Husky runs before commit (`.husky/pre-commit`):

1. `bun fmt` (format all files)
2. `bun lint` (fail on errors)
3. `bun audit --prod` (security check)
4. `bun run test:unit` (unit tests, must pass)
5. `bunx knip` (unused dependency check)
6. `bun i --frozen-lockfile` (verify lockfile consistency)

Commits fail if any step fails.

## CI Workflow

`.github/workflows/ci.yaml` on every push:

1. Checkout, install Bun
2. `bun i --frozen-lockfile`
3. `bun run build` (compiles web)
4. Install `mkcert`, generate HTTPS certs
5. Start server in background: `bun run --cwd ./packages/server dev`
6. Wait for `https://localhost:8080`
7. `bun run test:ci` (all tests, server must be live)

If server doesn't start or tests hang, check `packages/server/certs/` and evlog output.

## Schemas & Validation

Uses **Valibot** for runtime validation (not Zod):

- Main schema: `shared/src/schema.ts` (re-exported as `"shared/schema"`)
- Elysia server uses Valibot for request/response validation
- OpenAPI docs at `https://localhost:8080/openapi`

## Server Details

- **Framework**: Elysia with TypeScript AOT/precompile enabled
- **Entry**: `packages/server/src/api/server.ts`
- **Dev mode**: `DEV=true bun run --watch ./src/api/server.ts`
- **Data**: JSON files in `rawData/` (products, clothing)
- **Logging**: evlog setup in `packages/server/src/utils/logger.ts`
- **Plugins**: orders, products, search, static file serving

## Web Details

- **Framework**: Vite with raw TypeScript (no SPA framework)
- **Entry points**: `packages/web/src/*.html` (one per route)
- **Build**: `vite build` outputs to `packages/web/dist`
- **Imports**: `#pages/*`, `#utils/*`, `#data/*` (path aliases in `package.json:6-10`)

## Knip Configuration

Unused dependency detection is strict. Entry points per workspace:

- `packages/server`: `src/api/index.ts`
- `packages/web`: `src/scripts/pages/*.ts`
- Tests: `**/*.test.ts`

Ignored: `@types/trusted-types`, `husky`, `@socketsecurity/bun-security-scanner`.

## TypeScript Config

- `target: esnext`, `module: esnext`
- `allowImportingTsExtensions: true` (import `.ts` directly)
- `moduleResolution: bundler`
- No emit (`noEmit: true`)
- Strict mode enabled
- Allow synthetic default imports for compatibility

## Environment & Secrets

- **Dev mode** detected by `DEV=true` env var (used in server)
- **Prod mode** uses `Bun.env.PROD`
- SSL certs in CI: `packages/server/certs/` (must exist, generated by mkcert)
- No `.env` file pattern; config via `shared/config/constants.ts`

## Workspace Dependencies

Use workspace protocol: `"package": "workspace:*"` (always semver-compatible).
Packages are strict about exports; only use public API.

- `server` exports: `"."` (main API), `"./rawProducts"`, `"./clothing"`
- `web` and `shared` export specific modules under `"./module-name"`

## Common Pitfalls for Agents

1. **Bun, not npm**: `npm install` will fail; use `bun i`
2. **Barrel files banned**: No `index.ts` with `export * from '...'`
3. **Line limit 180**: Split long files; oxlint enforces this
4. **Three test projects**: Test must run in correct environment; `vitest.config.ts` assigns by path
5. **Server must run for CI tests**: `test:ci` expects `https://localhost:8080` live
6. **Valibot only**: No Zod; learn Valibot's API
7. **Max nesting 3 in tests**: Flatten deeply nested `describe()` calls
8. **Type imports**: Always use `import type` for types, `import` for values
9. **No cycles allowed**: Max depth 3; refactor if you hit this
10. **Pre-commit blocks pushes**: If tests fail locally, fix them before commit
